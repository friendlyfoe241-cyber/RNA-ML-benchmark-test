# Results Documentation

This folder contains all output files generated by the ML benchmark.

---

## CSV Files

| File | Description |
|------|-------------|
| `nested_cv_summary.csv` | **Nested Cross-Validation Results** - Contains averaged performance metrics across all outer CV folds. Uses inner CV for hyperparameter tuning. More robust estimate that prevents data leakage. Metrics are averaged over multiple train/test splits. |
| `nested_cv_fold_results.csv` | **Per-Fold Nested CV Results** - Detailed metrics for each individual fold of the nested cross-validation. Shows how performance varies across different data splits. |
| `holdout_test_results.csv` | **Holdout Test Set Results** - Final model performance evaluated on the held-out test set (20% of data). This is the single final evaluation on unseen data. Different from nested CV because it's evaluated once on the full trained model. |
| `holdout_predictions.csv` | **Model Predictions on Test Set** - Contains actual vs predicted labels for each sample in the holdout test set, along with predicted probabilities. Useful for debugging and detailed analysis. |

---

## PNG Files (Visualizations)

| File | Description |
|------|-------------|
| `ms_research_score_leaderboard.png` | Bar chart comparing all models by their MS Diagnostic Score. Models are ranked from highest to lowest score. Useful for quick visual comparison. |
| `holdout_metrics_heatmap.png` | Heatmap showing all metrics across all models. Rows are models, columns are metrics. Color intensity indicates performance level. |
| `holdout_roc_curves.png` | ROC (Receiver Operating Characteristic) curves for all models on the holdout test set. Shows the trade-off between true positive rate and false positive rate. AUC-ROC values are included in the legend. |

## Understanding the Difference: Nested CV vs Holdout Test

| Aspect | Nested CV (`nested_cv_summary.csv`) | Holdout Test (`holdout_test_results.csv`) |
|--------|-------------------------------------|------------------------------------------|
| **Evaluation Method** | Multiple train/test splits (5 outer folds) | Single train/test split |
| **Hyperparameter Tuning** | Inner CV within each fold (leakage-safe) | Uses pre-tuned parameters |
| **Metrics Shown** | Averaged across folds | Single evaluation on final model |
| **Use Case** | Research/publication (more rigorous) | Quick estimate of real-world performance |
| **Data Leakage** | None (designed to prevent it) | Minimal if preprocessing is done correctly |

**For publication:** Use `nested_cv_summary.csv` metrics as they are more scientifically valid.

**For quick validation:** Use `holdout_test_results.csv` for a single real-world estimate.

## CSV Column Descriptions

| Column | Description |
|--------|-------------|
| `Model` | Name of the machine learning algorithm |
| `Accuracy` | Proportion of correct predictions (TP + TN) / Total |
| `Balanced_Accuracy` | Average of sensitivity and specificity |
| `Sensitivity` (Recall) | True Positive Rate: TP / (TP + FN) |
| `Specificity` | True Negative Rate: TN / (TN + FP) |
| `Precision` | Positive Predictive Value: TP / (TP + FP) |
| `F1` | Harmonic mean of precision and recall |
| `MCC` | Matthews Correlation Coefficient |
| `AUC_ROC` | Area Under the ROC Curve |
| `PR_AUC` | Area Under the Precision-Recall Curve |
| `Brier` | Brier Score (calibration quality, lower is better) |
| `MS_Research_Score` | Clinically weighted composite score (0-100) |
| `AUC_95CI` | 95% Bootstrap Confidence Interval for AUC |
| `Sensitivity_95CI` | 95% Bootstrap Confidence Interval for Sensitivity |
| `Specificity_95CI` | 95% Bootstrap Confidence Interval for Specificity |
| `Runtime_sec` | Training and evaluation time in seconds |
| `Outer_CV_Folds` | Number of outer cross-validation folds |
| `Inner_CV_Folds` | Number of inner cross-validation folds |
